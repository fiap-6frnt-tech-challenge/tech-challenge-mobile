## Contexto

Garante que os fluxos da demo não quebram. **Maestro** é o mais simples para RN/Expo (YAML declarativo). Opcional — se o tempo apertar, cobrir por checklist manual (Task 07).

## Fluxos (Maestro flows)

1. **Auth:** abrir → registrar → logout → login
2. **Transação:** nova transação com sugestão de categoria → aparece na lista
3. **Filtro:** aplicar filtro por categoria → lista filtra
4. **Scroll infinito:** rolar → carrega mais
5. **Anexo:** anexar imagem → aparece (pode exigir device real)

```yaml
# .maestro/auth.yaml
appId: com.bytebank.mobile
---
- launchApp
- tapOn: "Criar conta"
- inputText: "Nome"
# ...
- assertVisible: "Dashboard"
```

## Gotchas

1. **Maestro** roda em emulador/device; uploads de câmera são difíceis de automatizar — cobrir anexo por galeria ou manualmente.
2. Usar um usuário de teste dedicado no Firebase p/ não poluir dados de demo.
